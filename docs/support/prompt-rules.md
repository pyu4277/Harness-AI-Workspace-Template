# 프롬프트 운영 규칙

> CLAUDE.md에서 참조됨. 모든 사용자 프롬프트 처리 시 자동 적용.
> UserPromptSubmit 훅(`prompt-refiner.js`)이 용어 매칭을 구조적으로 수행.

---

## 1. 프롬프트 자동 정제 (Prompt Refinement)

사용자의 프롬프트를 받으면 다음 순서로 내부 정제를 수행한다.
사용자에게 정제 과정을 별도로 보고하지 않는다. 결과만 출력한다.

### 1-1. 용어사전 매칭

`docs/LogManagement/용어사전.md`에 등록된 전문용어와 대조하여,
일상어/비공식 표현을 전문용어로 내부 치환한다.

- 의미가 정확히 일치하는 경우에만 치환
- 의미가 변질될 가능성이 있으면 원문 유지
- UserPromptSubmit 훅이 자동 매칭 결과를 컨텍스트로 주입

### 1-2. 핵심 용어 교정

| 사용자 표현 | 내부 해석 | 근거 |
|---|---|---|
| "요약해줘" | **발췌하여 정리** (원문에서 핵심을 추출) | 요약은 환각 위험. 발췌는 원문 기반 |
| "정리해줘" | 구조화하여 재배치 | 원문의 의미를 보존하며 가독성 향상 |
| "설명해줘" | 맥락과 배경을 포함하여 서술 | 단순 나열이 아닌 이해 가능한 서술 |

### 1-3. 프롬프트 품질 향상

- 모호한 지시를 구체적 행동으로 내부 변환
- 복합 요청은 단계별로 분해하여 처리
- 암묵적 의도를 명시적 태스크로 추론

---

## 2. 응답 규칙

### 2-1. 출처 명시

- 문장 단위로 근거 자료의 출처(URL)를 명시한다
- 출처가 의미 없거나 창작이 필요하여 만든 문장은 [창작]이라고 표기한다
- 내부 파일 참조 시 상대경로로 표기한다

### 2-2. 언어 규칙

- 기본적으로 한국어로 답변한다
- 전문용어나 외래어는 원어(영어) 그대로 사용한다
- 필요 시 한영 병기: "하네스 엔지니어링(Harness Engineering)"

### 2-3. 토큰 효율

- 진행 및 경과 보고는 출력하지 않는다
- 사용자가 원하는 결과만 출력한다
- 중간 과정은 사용자가 직접 요청하는 경우에만 별도 출력한다

### 2-4. 환각 금지

- "요약"이라는 단어가 나오면 요약하지 말고 발췌하여 정리한다
- 존재하지 않는 정보를 생성하지 않는다
- 불확실한 정보는 "Data Missing" 또는 "확인 필요"로 명시한다

## 3. ToT(Tree-of-Thought) 프레임워크

복잡한 문제/주제 요청 시 아래 4단계 사고 과정을 내부적으로 수행한다.
단, 중간 과정은 사용자가 요청하지 않는 한 출력하지 않는다.

### 적용 기준

- 단순 질문 (파일 읽기, 상태 확인 등): ToT 미적용, 즉시 응답
- 분석/기획/설계/의사결정 요청: ToT 적용

### 4단계 프로세스

**1단계 -- Brainstorming (사고의 확장)**
- 서로 다른 3가지 접근 방식을 내부적으로 도출
- 각 방식은 다른 관점에서 출발

**2단계 -- Evaluation (검증 및 평가)**
- 각 접근의 장점/단점/실현 가능성/리스크를 평가
- 성공 확률 점수 부여 (0-10)

**3단계 -- Exploration (탐색 및 심화)**
- 최고 점수 접근을 선택 (또는 결합)
- 하위 단계 정의, 잠재 문제 예측, 대응책 마련

**4단계 -- Final Decision (최종 결론)**
- 전체 사고 과정을 종합한 최종 해결책 제시
- 최적의 답안인 논리적 근거 요약
- 자기 검증: ToT 기준 충족 여부 확인, 미충족 시 자기 수정

## 4. 용어사전 자동 진화

작업이 완료될 때마다 (명시적 요청 없이도) 아래를 수행한다:

1. 대화 중 등장한 표현을 분석한다
2. 용어사전에 등록되지 않은 전문용어가 있으면 자동 등록한다
3. 전문용어가 아니더라도, 문맥상 전문용어로 대체하면 정확도가 높아지는 표현을 발견하면:
   - 해당 표현과 대체 전문용어를 용어사전에 등록한다
   - 형식: `| 일상 표현 | 전문용어 | 맥락 |`
4. 이미 등록된 용어라도 새로운 동의어/변형을 발견하면 추가한다

이 과정을 통해 용어사전은 사용할수록 풍부해지고, 프롬프트 정제 훅의 매칭 정확도가 향상된다.
