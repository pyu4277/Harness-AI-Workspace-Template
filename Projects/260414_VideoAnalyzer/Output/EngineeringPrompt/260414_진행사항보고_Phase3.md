## 문서별 핵심 내용

1. **도메인 정의서**: 14개 핵심 용어 정의, 파이프라인 5 Stage 용어, 도메인 개념도 Mermaid
2. **사업 기획서**: 프로젝트 목표 5개, 범위 IN 8개/OUT 5개, 리스크 5개+완화 전략, 도구 목록 12개
3. **요구사항 정의서**: FR 20개 (Stage별), NFR 7개, 하네스 강제 규칙 6개, 우선순위 매트릭스
4. **아키텍처 설계서**: 4계층 구조(입력/파이프라인/작업/출력), 보안 경계, ADR 3건, 데이터 흐름 시퀀스
5. **개발 표준 정의서**: 네이밍 규칙, Python 필수/금지 패턴, config.json 표준, 에이전트 운영 규칙 6개
6. **SRS**: 5 Stage 상세 기술 사양, 파라미터 범위/기본값, JSON 스키마, 에러 처리 6종
7. **상세 설계서**: 8개 모듈 함수 시그니처, 알고리즘, TransTest 참조점 2개, 모듈 의존성 체계
8. **순서도 및 절차도**: 전체 파이프라인 흐름도, Stage별 5개 상세 절차도, 에러 처리 절차도
