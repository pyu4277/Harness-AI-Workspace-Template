## 1. 파일명 규칙

| 종류 | 형식 | 예 |
|---|---|---|
| 회차 폴더 | `round_N` (N은 정수) | `round_1`, `round_12` |
| 리서치 파일 | `YYMMDD_[주제]_[태그].md` | `260414_삼성전자_IR.md` |
| 출력물 | `자소서.md` `이력서.md` `포트폴리오.md` (고정명) | — |
| 기업 프로파일 | `company_profile.md` (고정명) | — |

## 2. 자소서 문장 규칙 (STAR/CAR 강제)

### 글자수 200자 초과 문항 → STAR 4단 필수
```
[S] 어떤 상황/배경이었는가 (1-2문장)
[T] 무엇을 해야 했는가 (1문장)  
[A] 무엇을 했는가 (2-3문장, 주어=나)
[R] 결과/배움은 무엇인가 (1-2문장, 가능하면 수치)
```

### 200자 이하 문항 → CAR 3단
```
[C] 핵심 도전
[A] 행동
[R] 결과
```

### 지원동기 → PREP
```
[P] 결론: 왜 이 회사인가
[R] 근거: IR/ESG/최근사업에서 발견한 2가지 이상
[E] 사례: 내 경험/자료와의 교차점
[P] 재확인: 입사 후 기여 방향
```

## 3. 근거 인용 표기

모든 수치/성과/주장은 다음 형식으로 링크:

```markdown
[^ev1]: evidence_vault/03_자격증/260301_정보처리기사.pdf
[^rs1]: research_cache/round_2/01_합격수기/260401_삼성전자_SW_합격자.md
```

본문: `정보처리기사 취득 후[^ev1] 3개월간 알고리즘 학습 …`

## 4. 금지 표현 (훅 차단 대상)

| 패턴 | 이유 | 허용 조건 |
|---|---|---|
| "최고의", "최상의", "최대의" | 과장 | 공식 수치 근거 + 인용 |
| "모든 ~", "항상", "완벽히" | 절대 단언 | 없음 (금지) |
| "업계 1위", "역대급" | 근거 없는 시장 단언 | 공식 IR/공시 인용 시만 |
| "열정", "최선", "노력" (단독) | 진부 표현 | 정량 증거와 함께 |
| "감히", "외람되지만" | 자기비하 | 금지 |
| "~에 불과", "미천한" | 자기비하 | 금지 |

## 5. 작성 금기

- 타인 자소서 문장 복제 (합격 수기는 **구조/패턴만** 학습, 문장 복제 금지)
- 경력/경험 **창작** (evidence_vault/ 외부 사실 생성 금지)
- 날짜/수치 **추정** (evidence에 없으면 명시)
- 기업명/직무명 **오기** (input/meta.json 기준)

## 6. 에이전트 운영 규칙

- **작성 에이전트(writer)** 와 **검토 에이전트(reviewer)** 는 반드시 **다른 세션**
- 리서치(researcher)는 사용자 승인 하에 MCP 호출 (Exa/firecrawl 비용)
- 아이디어(ideator)는 evidence_vault/ + common/합격패턴_라이브러리 둘 다 읽은 후 제안
- 구현 완료 후 컨텍스트 40% 이하 — 초과 시 회차 분할
