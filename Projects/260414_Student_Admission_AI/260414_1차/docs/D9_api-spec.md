# D9. API 명세서 (MCP 인터페이스)

외부 통신은 모두 MCP 서버 경유. HTTP 직접 호출 금지.

## 하네스 엔지니어링 적용
| 기둥 | 역할 |
|------|------|
| 기둥1 | MCP 사용 경계 CLAUDE.md 명시 |
| 기둥2 | MCP 호출 시 PII 전송 훅 차단 |
| 기둥3 | MCP별 허용 에이전트 분리 |
| 기둥4 | MCP 실패 기록 → 캐시 전략 개선 |

## 사용 MCP 인벤토리

### 1. WebSearch (내장)
- **용도**: 합격 수기, 최신 기업 뉴스, 시사 맥락
- **호출 주체**: researcher 전용
- **금지 파라미터**: 학생 실명, 생년월일, 학번, 이메일, 전화번호
- **캐시**: research_cache/round_N/02_시사맥락/

### 2. WebFetch (내장)
- **용도**: 합격자 블로그, 기업 공식 채용 페이지, 뉴스 기사
- **호출 주체**: researcher 전용
- **주의**: robots.txt 준수, 저작권 있는 긴 인용 금지
- **출력**: 요약 + 원본 URL

### 3. Exa Search (mcp__exa-web-search__*)
- **용도**: 심층 리서치 (합격 수기 클러스터, 산업 동향)
- **호출 주체**: researcher 전용
- **쿼리 템플릿**:
  - `[기업명] [직무] 합격 자소서 공유`
  - `[산업] 2026 채용 트렌드`
  - `[기업명] 인재상 지원동기`
- **비용 인식**: 회차당 최대 10회

### 4. firecrawl (mcp__firecrawl__*)
- **용도**: 기업 IR 자료, 지속가능경영(ESG) 보고서 파싱
- **호출 주체**: researcher 전용
- **대상**: 공식 IR 페이지, 공시 시스템
- **출력**: research_cache/round_N/03_기업IR/, 04_ESG/

## 호출 규약

모든 MCP 호출 전 pre_tool_guard가 검증:
1. 호출 파라미터에 evidence_vault/ 경로가 포함되어 있는가? → 차단
2. 파라미터에 일반적 PII 패턴(한글이름+전화번호)이 감지되는가? → 차단
3. 현재 에이전트가 해당 MCP 호출 권한이 있는가? → 미권한 시 차단

## 리서치 결과 포맷

각 리서치 결과 파일:
```markdown
---
source_type: WebSearch | WebFetch | Exa | firecrawl
query: "..."
fetched_at: 2026-04-14
---

## 핵심 요약 (3-5줄)
...

## 인용 가능 구간
> "..." — [출처](URL)

## 원본 링크
- URL1
- URL2
```

## 실패 대응

| 실패 | 처리 |
|---|---|
| MCP 서버 미연결 | summary.md에 "리서치 미실행" 기록 + 사용자 알림 |
| 쿼리 0건 | 쿼리 재구성 후 재시도 (최대 2회) |
| 저작권 의심 | 요약 + 링크만, 본문 저장 금지 |
