# D2. 사업 기획서

## 하네스 엔지니어링 적용
| 기둥 | 역할 |
|------|------|
| 기둥1 | CLAUDE.md에 WHY 3줄 요약 |
| 기둥2 | 스코프 OUT 항목 훅 차단 |
| 기둥3 | evidence_vault/ 외 자료 인용 시 차단 |
| 기둥4 | 회차별 결과를 KPI 대비 자동 평가 |

## 1. 개요 (WHY)

학생 한 명이 합격할 때까지 수십 개 기업에 **매번 처음부터** 자소서를 쓰는 비효율을 해소한다. 객관 자료를 단 한 번만 구조화하면, 매 회차마다 **기업 맞춤 + 최신 시사 반영 + 과거 회차 개선점 반영**된 지원서가 생성되게 한다.

## 2. 목표 (WHAT)

- 학생 **객관 자료 1회 등록**으로 N개 기업 지원서 자동 초안 생성
- 회차 반복 시 **개선이력 누적** → 회차 거듭할수록 품질 상승
- 기업 **공식 문서 기반 맞춤화** (IR/ESG/인재상)
- **허위사실 0건** — 모든 주장은 evidence_vault/ 근거 링크

## 3. 범위

### IN (포함)
- 자소서 / 이력서 / 포트폴리오 작성 초안 생성
- 기업 프로파일 자동 수집 + 리서치
- 회차 관리 (1차, 2차, …)
- 회차 간 개선이력 추적
- 최신 합격 사례 / 시사 / 국제정서 리서치
- 기발한 아이디어 제안 (사용자 자료 × 합격 패턴 교차 분석)

### OUT (제외)
- 실제 기업 지원 시스템 제출 (사용자 본인 수행)
- 허위 경력/자격 생성 (기둥3 차단)
- 타인 자소서 무단 복제 (저작권)
- 합격 보장 (확률적 보조 도구)

## 4. 이해관계자

| 역할 | 주체 | 관심사 |
|---|---|---|
| 학생 (사용자) | 지원자 본인 | 합격, 문장 품질, 시간 절약 |
| 객관 자료 | 학교/기관 발급 | 사실성, 일관성 |
| 기업 | 채용담당 | 진정성, 역량 일치, 허위 없음 |
| 사회적 맥락 | 언론/IR/통계 | 시의성, 산업 이해도 |

## 5. 핵심 리스크 + 대응

| 리스크 | 심각도 | 대응 (구조적) |
|---|---|---|
| 허위사실 자동 생성 | 치명 | post_tool_validate 훅이 evidence 링크 없는 수치 주장 차단 |
| 기업별 정보 오염 (전 회차 기업 정보가 새 회차에 섞임) | 높음 | 회차 불변성 + company_profile 회차별 독립 |
| 리서치 과다 수집으로 컨텍스트 40% 초과 | 중간 | research_cache/ 요약 캐시. 원문은 링크만 |
| 학생 개인정보(생기부) 유출 | 치명 | .env + evidence_vault/ 쓰기만 허용, 외부 MCP에 전달 금지 |
| 타인 자소서 무단 학습 | 중간 | 합격 수기는 공개 자료만, 인용 출처 의무 |

## 6. 하네스 적용 계획

- CLAUDE.md 60줄: 핵심 규칙 + 절대 금지 (허위/회차수정/PII전송)
- 훅 3종: pre_tool_guard(경로), post_tool_validate(금지 용어/허위), post_round_check(회차 완결성)
- permissions: evidence_vault/ 읽기만, round_N/ 쓰기, 이전 round_M(M<N) 쓰기 차단
- 피드백 루프: 각 회차 종료 시 CHANGELOG.md 자동 작성, 구조 개선 제안

## 7. KPI

| 지표 | 목표 |
|---|---|
| 회차당 자소서 생성 시간 | 1시간 이내 |
| 허위사실 발생 건수 | 0 |
| 회차 품질 상승률 | N차 > (N-1)차 (자기평가 + 사용자 평가) |
| 합격 수기 인용 건수 | 회차당 3건 이상 |
| 기업 공식 문서 근거 수 | 회차당 2건 이상 (IR/ESG/인재상) |
