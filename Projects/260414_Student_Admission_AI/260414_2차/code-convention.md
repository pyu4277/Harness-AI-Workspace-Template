# Code / 문서 규약

> CLAUDE.md의 "참조 문서"로 연결. 상세 기준은 이 파일.

## 1. 파일명

| 종류 | 규칙 |
|---|---|
| 회차 폴더 | `round_N` (N=정수, 0 시작 금지) |
| 리서치 파일 | `YYMMDD_[주제]_[태그].md` |
| 설계문서 | `D[번호]_[kebab-case].md` (docs/) |
| 출력 고정 | `자소서.md` `이력서.md` `포트폴리오.md` |
| 기업 프로파일 | `company_profile.md` (고정) |
| 개선이력 | `CHANGELOG.md` (고정) |
| 아이디어 | `ideas.md` (고정) |
| 입력 메타 | `meta.json` |

## 2. Markdown 스타일

- 최상단 `# 제목` 1개
- `## 섹션` (H2) 기준, H4 이하 지양
- 표 > 긴 문단 (가독성)
- 흐름은 Mermaid 블록 (```mermaid)
- 인용 각주는 `[^태그]` 사용, 하단에 `[^태그]: 경로` 정의

## 3. 인용 규약

| 접두어 | 출처 |
|---|---|
| `[^ev_X]` | evidence_vault/ 내 파일 |
| `[^rs_X]` | research_cache/round_N/ 내 파일 |
| `[^ext_X]` | 외부 URL (연구용 참고, 자소서 본문 인용 자제) |

## 4. 자소서 문장 체크

- 200자 초과 → STAR 4단
- 200자 이하 → CAR 3단
- 지원동기 → PREP
- 모든 수치/성과 → `[^ev_X]` 또는 `[^rs_X]` 링크
- 주어는 "저" 또는 "제가" (관찰자 시점 금지)
- 과거형 + 현재 완료 혼용 금지 (과거형 통일)

## 5. 금지 표현 사전 (요약)

전체 목록은 `common/금지표현_사전.md`. 대표:
- 과장: "최고의", "완벽한", "역대급", "업계 1위" (근거 없이)
- 절대: "모든", "항상", "완전히"
- 자기비하: "~에 불과", "감히"
- 진부: "열정", "최선", "노력" (단독 사용)

## 6. Git 커밋

- 형식: `<type>: <description>` (feat/fix/docs/chore)
- 본문: 이모티콘 금지, 절대경로 금지
- `git add -A` 지양 — 개별 파일 add

## 7. 스크립트 코딩

- Python 3.11+
- subprocess 사용 시 `errors="replace"`
- 절대경로 하드코딩 금지 — `Path(__file__).parent` 또는 환경변수

## 8. 훅 응답 규약

- `exit 0` = 통과
- `exit 1` = 비치명 오류 (진행 허용)
- `exit 2` = 차단 (stderr에 사유 명시)
- 경고(WARN)는 stderr에 `[WARN]` 접두
