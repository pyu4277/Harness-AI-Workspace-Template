# 삼성전자 MX사업부 SW직무 서류 + SW역량테스트 후기 (구조 요약)

## 자소서 4문항 우선순위 (합격자 인식)
**개발 관련 경험 >> 지원동기 = 성장 과정 >> 사회 이슈**

문항별 접근:
- **지원동기**: 학과 경험을 사업부 업무와 연결하는 직선형 서사
- **성장 과정**: SW 개발 관심이 시간순으로 발전한 연대기형
- **사회 이슈**: 전통적 서론-본론-결론 구조 (가장 적은 가중치)
- **개발 경험**: "개발 중 직면한 문제와 해결 접근법" 중심 -- STAR 변형

## 자소서 운영 팁
- 서류 한 번 통과하면 동일 자소서 재제출도 통과 가능 -- 차별화보다 기본 자격 충족 평가 비중이 큼을 시사
- 즉, **첫 통과 시점에 자소서를 안정화**하고 이후 전형(코테/면접)에 자원 집중

## SW 역량테스트
- 시간: 3시간 -> 4시간 확장
- 구성: 전통형 1문항 + B형(스켈레톤 코드 완성형) 1문항
- 언어: Python 보다 C++/Java 가 스켈레톤 형식에 적합
- 본인 결과: 1문제 BFS로 해결, 1문제 미해결 -> 면접까지 진출

## 일정 (참고용)
- 서류 합격: 4월 초
- 코테: 4월 9일
- 결과 발표: 5월 8일 (GSAT 합산 평가로 ~1개월 소요)

- "지원동기 및 포부"는 학과/연구 경험과 MX 사업부 업무의 직선 연결 서사
- "보완점"은 알고리즘/시간복잡도 같은 구체적 약점 + 극복 의지 (실제 코테 경험과 일치)
