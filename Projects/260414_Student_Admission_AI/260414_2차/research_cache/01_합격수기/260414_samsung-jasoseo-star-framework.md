# 삼성전자 SW 자소서 STAR 프레임 + 인재상 작성 가이드

## 직무 경험 문항 공식 형식
"프로그램 개발, 알고리즘 풀이 등 SW개발 관련 경험 중 가장 어려웠던 경험과 해결방안에 대해 구체적으로 서술"
필수 4요소: **과제 개요 / 어려웠던 점 / 해결 방법 / 결과**

## STAR 구조 매핑
| STAR | 자소서 활용 |
|---|---|
| Situation (상황) | 과제 개요, 배경, 제약 조건 |
| Task (과제) | 본인 역할, 책임 범위 |
| Action (행동) | 해결 방법, 사용 기술, 의사결정 이유 |
| Result (성과) | 정량 결과 (성능 개선 %, 문제 해결, 학습) |

## 인간적 자소서 만들기 (감점 회피 -> 가점 전환)
- 문장마다 "내 생각" 한 줄 추가 -> 감정/통찰/결정 이유 노출
- 사실 나열형이 아닌 "왜 그렇게 판단했는가" 가 인재상 평가 기준에 부합

## 평가 가중치 (지원동기 문항)
- 지원동기 = 회사의 **최근 이슈 인지도** 평가
- 즉 "Galaxy S26 / 트라이폴드 / 에이전틱 AI / 신년사" 같은 2026년 이슈와 본인 동기를 직접 연결해야 함

## writer 에이전트 활용 체크리스트
- [ ] 4문항 모두 STAR 4요소 명시적 내포
- [ ] "내 생각/결정 이유" 한 줄을 최소 문항당 2개
- [ ] 지원동기에 2026년 삼성 최근 이슈 (Galaxy S26 + 에이전틱 AI + 트라이폴드) 1개 이상 직접 인용
- [ ] 결과는 가능하면 정량 (% / 시간 / 사용자 수)
