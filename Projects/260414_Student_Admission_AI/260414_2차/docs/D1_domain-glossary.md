# D1. 도메인 정의서 + 용어사전

> 자소서/이력서/포트폴리오 작성에 쓰이는 모든 용어를 통일한다.

## 하네스 엔지니어링 적용
| 기둥 | 이 문서에서의 역할 |
|------|-------------------|
| 기둥1 (컨텍스트) | CLAUDE.md 요약본에 용어 Top-20 참조 링크 |
| 기둥2 (CI/CD) | 훅이 금지 용어(과장 표현) 자동 차단 |
| 기둥3 (도구경계) | evidence_vault/ 외 자료로 "사실" 주장 금지 |
| 기둥4 (피드백) | 회차별로 새 용어 발견 시 자동 추가 |

## 도메인 개요

학생(초·중·고 ~ 대학생) 개인 이력을 기반으로 **기업별 맞춤 지원서**(자소서/이력서/포트폴리오)를 **회차 반복**으로 합격할 때까지 개선하는 도메인.

## 핵심 개념

| 한글 | 영문 | 정의 |
|---|---|---|
| 자기소개서 | Personal Statement (PS) | 성장배경·지원동기·입사후포부 등을 서술하는 정성 문서 |
| 이력서 | Résumé / CV | 학력·경력·자격·어학을 정형화해 나열한 문서 |
| 포트폴리오 | Portfolio | 프로젝트 결과물 + 역할 + 회고를 묶은 시각 자료 |
| 객관 자료 | Evidence | 생활기록부·성적·자격증·수상내역 등 **증명 가능한 사실** |
| 회차 | Round | 1차·2차 지원서 세트 (`round_N/`) |
| 개선이력 | Changelog | 회차 간 변경점 + 이유 (`CHANGELOG.md`) |
| 기업 프로파일 | Company Profile | 인재상·직무·최근이슈 요약 (`company_profile.md`) |
| 직무기술서 | Job Description (JD) | 기업이 제시한 모집 요강 원문 |
| NCS | National Competency Standards | 국가직무능력표준 — 직무별 능력단위 |
| 역량 | Competency | 업무 수행에 필요한 지식·기술·태도의 결합 |
| 핵심성과 | Key Achievement | 정량 지표로 증명 가능한 성과 |

## 작문 기법 용어

| 약어 | 풀이 | 적용 |
|---|---|---|
| STAR | Situation-Task-Action-Result | 경험 서술의 표준 4단 구조 |
| CAR | Challenge-Action-Result | STAR의 축약형. 짧은 문항에 사용 |
| PREP | Point-Reason-Example-Point | 논리 전개 구조. 지원동기에 적합 |
| 5W1H | What/Why/Who/When/Where/How | 사실 기반 서술의 최소 요소 |

## 리서치 개념

| 한글 | 영문 | 정의 |
|---|---|---|
| 합격 수기 | Success Case | 해당 기업·직무에 실제 합격한 사람의 공개 자소서/후기 |
| 시사 맥락 | Macro Context | 경제·사회·국제 정세 중 지원 산업과 연결되는 흐름 |
| IR 자료 | Investor Relations | 기업 공식 재무·전략 문서 — 지원동기 근거 |
| ESG 보고서 | ESG Report | 지속가능경영 보고서 — 인재상/가치관 근거 |

## 금지 용어 (허위사실 / 과장)

아래 표현은 훅(post_tool_validate.sh)이 자동 차단:
- "최고의", "완벽한", "탁월한" (객관 증거 없으면)
- "모든 것을 바쳐", "혼자 해냈다" (근거 없는 단독 성과 주장)
- "~에 불과" (자기 비하 후 과장 패턴)
- "업계 1위", "역대급" (수치 근거 없이)

## 회차 전용 용어

| 용어 | 의미 |
|---|---|
| 회차 복제 | 이전 회차 폴더를 그대로 복사 후 새 회차로 시작 |
| 구조 개선 | 이전 회차 수행 중 발견한 폴더/템플릿 구조의 문제를 차기 회차에 반영 |
| 불변성 | 이전 회차 파일은 수정 금지 — 개선은 신규 회차에서만 |
| 기발한 아이디어 | 합격 사례 패턴과 사용자 자료의 **비자명한 교차점** — 에이전트 제안 슬롯 |
