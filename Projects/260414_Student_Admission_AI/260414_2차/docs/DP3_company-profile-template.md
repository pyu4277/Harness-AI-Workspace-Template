# D+3. 기업 프로파일 템플릿

각 회차의 `company_profile.md`를 작성할 때 쓰는 표준 템플릿 + 슬롯 정의.

## 하네스 엔지니어링 적용
| 기둥 | 역할 |
|------|------|
| 기둥1 | writer가 이 템플릿을 반드시 참조 |
| 기둥2 | 슬롯 누락 시 훅이 경고 |
| 기둥3 | 외부 자료 참조는 research_cache/에서만 |
| 기둥4 | 슬롯 패턴 개선은 CHANGELOG 반영 |

## 슬롯 구조

company_profile.md는 다음 10개 슬롯으로 구성:

```markdown
# [기업명] 기업 프로파일 — round_N

## S1. 기본 정보
- 설립연도: [IR 인용]
- 본사: [공식 홈페이지]
- 산업 분류: [DART 공시 기준]
- 주력 제품/서비스: ...

## S2. 최근 3년 실적 요약
- 매출: [^rs_ir_1]
- 영업이익: [^rs_ir_1]
- 성장 동력: [최근 분기 컨콜 발췌]

## S3. 신사업 / 전략 방향
- 공시 자료 기반 3개 이내 [^rs_ir_2]
- 내 경험/자료와 연결 가능한 것 표시 [^ev_*]

## S4. 인재상 (공식 문구)
- 채용 페이지 원문: "..." [^rs_esg_1]
- 3-5개 키워드 추출
- 각 키워드에 매칭되는 내 evidence 표시

## S5. ESG / 가치관
- 지속가능경영보고서 핵심 문구 [^rs_esg_1]
- 대표 CSR / 사회 기여 활동

## S6. 최근 이슈 / 시사 맥락
- 최근 6개월 뉴스 3건 [^rs_news_*]
- 내 지원과 연결되는 흐름

## S7. 직무 요구사항 (JD 정리)
- 필수 역량: (input/job_description.md 정제)
- 우대 사항
- 각 항목별 내 매칭 근거 [^ev_*]

## S8. 경쟁사 대비 차별점
- 내가 이 회사를 '선택한' 이유 근거
- 경쟁사 대비 강점 2-3건

## S9. 지원 동기 가설 (writer가 자소서에 사용할 프레임)
- 3-5문장 요약
- 모든 주장에 [^rs_*] 또는 [^ev_*] 링크

## S10. 주의사항 / 리스크
- 민감 이슈 (회피할 화제)
- 면접 예상 질문 (차기 활용)

## 인용 목록
[^rs_ir_1]: research_cache/round_N/03_기업IR/...
[^rs_ir_2]: ...
[^rs_esg_1]: ...
[^rs_news_1]: ...
[^ev_1]: evidence_vault/...
```

## 슬롯 작성 규칙

| 슬롯 | 필수 | 최소 근거 수 |
|---|---|---|
| S1 | 필수 | 1 |
| S2 | 필수 | 1 (IR) |
| S3 | 필수 | 2 (IR) |
| S4 | 필수 | 1 (채용 페이지 or ESG) |
| S5 | 선택 | 1 |
| S6 | 필수 | 2 (뉴스) |
| S7 | 필수 | input/JD |
| S8 | 선택 | 1 |
| S9 | 필수 | 3 (혼합) |
| S10 | 선택 | 0 |

필수 슬롯 누락 시 post_tool_validate 훅이 경고.

## 근거 없는 작성 금지

- S2의 수치는 반드시 IR 인용 (추정 금지)
- S4 인재상은 공식 문구만 (가공 금지)
- S6 뉴스는 출처 링크 필수
- S9 동기는 S2~S6 근거들의 조합으로만 구성

## 회차 간 비교

동일 기업 재지원 시 이전 회차 company_profile.md와 diff:
- S6(최근 이슈)만 업데이트 대상
- 나머지는 큰 변화 없으면 "재사용" 태그
