# FileNameMaking

비정형 문서(논문, 보고서, 규정 등)를 지정된 평가 주제로 딥리딩(Deep Reading)하여 0~100점 스코어를 산출하고, 사용자가 정의한 파일명 양식으로 일괄 변경하는 3단계 파이프라인.

보조 스크립트: `scripts/generic_evaluator.py`

---

## 3단계 실행 파이프라인

### Step 1: 필수 입력값 수집 (Handshake)

스킬 활성화 시 즉시 다음 3가지 정보를 요청합니다:

```
FileNameMaking 스킬이 활성화되었습니다.

성공적인 파일명 변경을 위해 다음 3가지를 알려주세요:

1. 작업 위치 (Target Directory): 분석할 파일들이 있는 폴더 경로
   예: D:\Data\Papers

2. 파일명 양식 (Naming Format): 최종 파일명 규칙
   예: [랭킹]_[연도]_[문서종류]_[실제제목].ext

3. 평가 주제 및 기준 (Evaluation Topic): 중요도 평가 기준 설명
   예: ESG 경영 및 탄소중립 실천 사례와 연관성이 높을수록 고득점
```

### Step 2: 파이프라인 실행 (Analysis & Report)

3가지 정보 수신 후 즉시 Python 스크립트를 작성하여 실행합니다.

**파이프라인 구성:**

1. **추출**: 폴더 내 파일(pdf, hwpx 등)에서 텍스트 추출. PyPDF2/pdfplumber 활용. 연도·제목은 첫 페이지 기반 보정.

2. **딥리딩 평가**: 평가 주제 기반으로 각 문서에서 [목적/목표], [방법/절차], [결론/시사점] 문장을 추출. 주제 연관성 점수(0~100) + 근거(Justification) 도출.

3. **리포트 생성**: `temp_analysis_report.md` 출력 — 매핑 테이블(No, 기존이름, 새이름, 점수) + 개별 정성 평가 근거.

4. **컨펌 요청**: 리포트 내용 제시 후 "이대로 일괄 변경을 진행할까요?" 확인.

### Step 3: 파일명 일괄 변경 (Finalize)

사용자 승인 후 매핑 테이블을 파싱하여 `os.rename()`으로 실제 파일명 변경. 변경 완료 후 성공률(Gap Analysis) 로그 출력.

---

## 제약사항

- 파일명 길이: `[실제제목]` 부분은 80자 이내로 자르는 로직 필수
- 불법 문자 치환: `\ / : * ? " < > |` → 언더스코어(`_`) 또는 생략
- 외부 API 없을 경우: 오프라인 NLP 휴리스틱으로 유의미한 점수 도출

- "논문들 파일 정리해줘, 양식은 [점수]_[연도]_[제목]"
