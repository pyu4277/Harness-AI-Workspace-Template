# Term Organizer

현재 대화에서 전문용어를 자동 추출하여 `docs/LogManagement/용어사전.md`에 누적 저장하는 용어 관리 스킬.

---

## 트리거 명령

| 명령 | 동작 |
|:---|:---|
| `전문용어 정리해줘` | 현재 대화 분석 → 신규 용어 추출 → 파일 저장 |
| `전문용어 추출해줘` | 장황한 표현 → 전문용어 대체 제안 |
| `전문용어 알려줘` | 용어사전 파일 경로 안내 |
| `전문용어 추가해줘: [용어]` | 지정 용어 1개 수동 추가 |

---

## 용어사전 파일 위치

```
${PROJECT_ROOT}/docs/LogManagement/용어사전.md
```

---

## 실행 절차

### "전문용어 정리해줘" 입력 시

1. **대화 전체 분석**: 현재 대화에서 전문용어 후보 추출
2. **기존 파일 확인**: `용어사전.md`를 읽어 이미 등록된 용어인지 확인
3. **신규 용어만 추가**: 중복 용어 스킵, 새 용어만 테이블 하단에 추가
4. **업데이트 기록 갱신**: 날짜/추가수량 업데이트
5. **완료 보고**: "N개 용어를 추가했습니다." 형식으로 응답

### "전문용어 추출해줘" 입력 시

장황하게 설명한 표현 중 정확히 대응되는 전문용어가 있을 경우:

```
'[원래 표현]'으로 말씀하신 내용은 '[전문용어]' 전문 용어로 대체할 수 있습니다.

해당 용어를 사전에 추가할까요?
```

**동작 기준:**
- 확실한 경우만 제안. 애매하면 제안하지 않음.
- 3개 이상 문장으로 설명한 개념이 전문용어 하나로 압축 가능한 경우 대상.
- `용어사전.md` 등록 용어 우선 참조, 없으면 일반 전문용어 지식 활용.

### "전문용어 알려줘" 입력 시

> 용어사전 파일:
> `${PROJECT_ROOT}/docs/LogManagement/용어사전.md`

---

## 컬럼 작성 기준

| 컬럼 | 작성 규칙 |
|:---|:---|
| 번호 | 기존 최대 번호 + 1 자동 부여 |
| 용어 | 한국어 공식 명칭 (없으면 원어 그대로) |
| 원어 | 영어 또는 원래 언어 표기 |
| 태그 | 1~3개 권장. `#AI` `#개발` `#통계` `#문서` `#데이터` 등 |
| 난이도 | 쉬움 / 보통 / 어려움 — 일반인 기준 |
| 쉬운 설명 | 비전문가도 이해할 수 있게, 일상 비유 포함 |
| 전문 설명 | 기술적으로 정확한 정의 |
| 예시 | 실제 문장이나 코드 한 줄 |
| 비고 | 관련 용어, 주의사항 (선택) |
| 등록일 | `YYYY-MM-DD` 형식으로 자동 입력 |
