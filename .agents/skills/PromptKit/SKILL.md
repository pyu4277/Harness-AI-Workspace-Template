# PromptKit — 프롬프트 변환 + 예시 생성 통합 스킬

모호한 요청 → 구조화된 프롬프트 변환(4단계) + 맥락 기반 예시 자동 생성을 단일 스킬로 처리합니다.

보조 스크립트: `scripts/example_integrator.py`

---

## 5단계 실행 파이프라인

### Step 1: 용어 인지 & 타당성 분석

- 사용자 요청의 핵심 개념과 전문 용어를 식별
- `docs/LogManagement/용어사전.md` 참조하여 기존 정의 용어와 일치
- 요청이 실현 가능한지 타당성 판단

### Step 2: 기법 분석 & 도구 오케스트레이션

- 프롬프트 목적 분류: 코드 생성 / 문서 작성 / 분석 / 창작 / 대화 등
- 적용 기법 선택: Few-shot, Chain-of-Thought, Role-play, Constraint 등
- 필요한 강제 도구 명시

### Step 3: 작성 & 각주 매핑 & 검증

- 선택된 기법을 적용하여 프롬프트 초안 작성
- 각 규칙/지침에 각주 번호 매핑 (근거 추적)
- `mdGuide` 스킬 기준으로 Zero-Defect 마크다운 검증

### Step 4: 사용자 승인 후 적용

- 변환된 프롬프트를 제시하고 승인 요청
- 승인 후 후속 스킬 자동 연계 제안

### Step 5: 예시 자동 생성 (맥락 기반)

현재 대화 전체를 분석하여 관련 예시를 즉시 생성합니다.

**예시 유형:**

| 유형 | 설명 | 형식 |
|:---|:---|:---|
| `code` | 코드 스니펫 또는 구현 예시 | 언어 지정 코드 블록 |
| `data` | 샘플 데이터 (JSON, CSV, 표) | 표 또는 코드 블록 |
| `scenario` | 실제 사용 사례 서술 | 번호 매긴 단계 |
| `comparison` | 대안 비교 | 표 (장단점 포함) |
| `before_after` | 변환 전후 비교 | diff 또는 쌍 블록 |
| `conversation` | 대화/상호작용 예시 | 인용 대화 형식 |
| `diagram` | 시각적 구조 예시 | Mermaid 또는 ASCII |
| `error_fix` | 문제→해결 쌍 | 문제 블록 + 해결 블록 |

**예시 생성 원칙:**
- 사용자에게 어떤 예시가 필요한지 묻지 않음 — 자율 분석 결정
- 최소 3개 생성 (맥락이 명확히 적은 수를 요구하는 경우만 예외)
- `foo/bar/baz` 같은 placeholder 금지 — 실제 프로젝트 맥락 사용

## 트리거 분기

| 요청 유형 | 실행 단계 |
|:---|:---|
| 프롬프트 변환/작성 요청 | Step 1~4 실행 |
| 예시 생성 요청 | Step 5 실행 |
| 프롬프트 변환 + 예시 필요 | Step 1~5 전체 실행 |

## example_integrator.py 연계

프롬프트 변환 결과에 자동으로 예시를 삽입할 때 사용:

```bash
python ".agents/skills/PromptKit/scripts/example_integrator.py" \
  --context "[현재 대화 맥락]" --type code
```
